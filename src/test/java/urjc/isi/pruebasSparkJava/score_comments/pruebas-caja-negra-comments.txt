 -- DESCRIPCIÓN DE LAS PRUEBAS DE CAJA NEGRA DE LA CLASE COMMENT --

1. Identificar los métodos a testar
	Método 1: newComment (String text, int user, String film, Injector I)
	Método 2: commentsFilm (String film, Injector I)
	Método 3: commentToString (String matrix_coment[][])
	Método 4: portComment (Injector I)

2. Determinar los parámetros a testar
	Método 1: 
		1: texto del comentario (String)
		2: número de usuario (int)
		3: nombre de película (String)
		4: injector a la base de datos (Injector)
	Método 2:
		1: nombre de la película (String)
		2: injector a la base de datos (Injector)
	Método 3: 
		1: conjunto de comentarios en una película (String)
	Método 4: 
		1: injector a la base de datos (Injector)

3. Elegir caracterizaciones y división en bloques

	Método 1:
		C1: texto del comentario no nulo => T/F => A1/A2
		C2: número de usuario válido (positivo) => T/F => B1/B2
		C3: nombre de película no nulo => T/F => C1/C2
		C4: injector válido => T/F => D1/D2
	Método 2:
		C1: nombre de película no nulo => T/F => A1/A2
		C2: injector válido => T/F => B1/B2
	Método 3: 
		C1: conjunto de comentarios en una película no nulo => T/F => A1/A2
	Método 4:
		C1: injector válido => T/F => A1/A2

4. Aplicar criterios de cobertura.

	Método 1: teniendo en cuenta que si cualquiera de las cuatro caracterizaciones es falsa, 
	no es necesario probar si el resto son o no verdaderas, nos es suficiente con 5 tests 
	(que cada una sea falsa + happy path)
		Test 1: texto del comentario nulo (FXXX => A2XXX)
		Test 2: número de usuario no válido (negativo) (XFXX => XB2XX)
		Test 3: nombre de película nulo (XXFX => XXC2X)
		Test 4: Injector no válido (XXXF => XXXD2)
		Test 5: happy path, ninguna entrada inválida (TTTT => A1B1C1D1)

	Método 2: tenemos dos caracterizaciones, y al igual que en el caso anterior, solo debemos
	comprobar si alguna de las dos ha fallado y si todas han ido bien. En total, serían 3 tests
		Test 1: nombre de película nulo (FX => A2X)
		Test 2: injector no válido (XF => XB2)
		Test 3: happy path, nombre de película no nulo e injector válido (A1B1)

	Método 3: en este caso, solo tenemos una caracterización, de manera que probaremos todos los
	posibles outcomes
		Test 1: conjunto de comentarios nulo (A2)
		Test 2: happy path, conjunto de comentarios no nulo (A1)

	Método 4: al igual que en el caso anterior, tenemos tan solo una caracterización, de manera que
		Test 1: injector no válido (A2)
		Test 2: happy path, injector válido (A1)

5. Traducir las combinaciones a valores que pasar a los tests en cuestión

	Método 1: 
		Test 1: (null, 4, "Kill Bill: Volumen 3", I)
		Test 2: ("comentario", -1, "Kill Bill: Volumen 3", I)
		Test 3: ("comentario", 4, null, I)
		Test 4: ("comentario", 4, "Kill Bill: Volumen 3", null)
		Test 5: ("comentario", 4, "Kill Bill: Volumen 3", I)

	Método 2:
		Test 1: (null, I)
		Test 2: ("Kill Bill: Volumen 3", null)
		Test 3: ("Kill Bill: Volumen 3", I)

	Método 3:
		Test 1: (null)
		Test 2: (["comentario 1", "comentario 2"])

	Método 4:
		Test 1: (null)
		Test 2: (I)
